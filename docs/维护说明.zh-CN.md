# 维护说明

这份文档是给以后继续维护这个项目用的。

## 1. 安卓 App 重新编译

```bash
cd /home/ak/树莓派/tp-satochip-signer
./gradlew :app:assembleRelease --console=plain
```

## 2. 树莓派签名器源码位置

- `pi-signer-py/bin/pi-signer`
- `pi-appliance/`
- `seedsigner-os/opt/rootfs-overlay/`

## 3. 当前固件的重打方式

当前不是每次都全量重编 Buildroot。

如果只是改：

- UI
- `pi-signer-py`
- overlay 里的脚本或资源

直接重打运行时镜像即可：

```bash
bash /home/ak/树莓派/tp-satochip-signer/scripts/repack_runtime_image.sh
```

这个脚本会：

1. 从现有基础镜像提取 `syscfg.dat`
2. 覆盖最新 `rootfs-overlay`
3. 重新生成新的 `system-update-*.img`

## 4. 固件发布

当前瘦身后的固件压缩包已经低于 GitHub 单文件限制，可直接保留为单文件：

```bash
cd /home/ak/树莓派/tp-satochip-signer/dist
sha256sum system-update-latest.img.xz > system-update-latest.img.xz.sha256
```

## 5. 安卓 App 发布文件

当前仓库里保留稳定文件名：

- `dist/tp-qr-relay-android-latest.apk`
- `dist/tp-qr-relay-android-latest.apk.sha256`

## 6. Git 提交建议

每次发布前至少检查：

```bash
git status
```

重点确认不要把这些推上去：

- `logs/`
- `*.deb`
- `local.properties`
- `seedsigner-os/buildroot_dl/`
- 时间戳版大镜像

## 7. 这套架构的关键点

1. 手机负责吃掉 TP 高频动态码。
2. 树莓派只扫低频静态中转码。
3. 私钥不在树莓派上，签名在智能卡内完成。
4. 树莓派只负责请求解析、PIN 输入、APDU 转发和结果二维码显示。
